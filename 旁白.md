## 第 16 页 · 怎么描述需求

**预估时长：2.5 分钟**

怎么描述需求？这是很多人刚开始用 AI 时的困惑。

其实不需要一次写完整的提示词。看左边这个对话过程：

我先说了一个大概：「制作一个 AI 图片社区的广场界面，顶部导航栏、灵感画廊内容页、要自适应、要有动效……有不清楚的地方请先向我提问。」

然后 AI 问了我 6 个问题：产品名称叫什么？图片是什么类型？图片来源用什么？展示多少张？点赞要不要记住状态？其他页面要不要做？

我一一回答后，AI 就整理出了右边这份完整的提示词——页面概述、页面结构、风格要求、交互动效、技术方案，全都有了。

所以关键技巧是：不要想着一次写完美，先说大概，让 AI 问你问题，最终的提示词是「聊」出来的。

## 第 26 页 · 提交即上线

**预估时长：2 分钟**

这是整个部署流程：代码提交 → GitHub → Netlify 自动构建 → 生成预览链接。

简单介绍一下这两个工具：

GitHub 是全球最大的代码托管平台，可以理解为代码的云盘，支持版本管理、多人协作。我们的原型代码就存在这里。

Netlify 是一个静态网站托管服务，它能自动监听 GitHub 的变化——你一提交代码，它就自动构建并生成一个可访问的链接。

所以整个流程是：你在本地改完代码，提交到 GitHub，Netlify 自动帮你部署，几十秒后就能通过链接访问最新版本。

把这个链接发给开发、发给老板，他们直接打开就能看效果，不需要安装任何软件。

初始配置（Git + GitHub + Netlify）网上资料很多，走一遍就熟悉了。

## 第 30 页 · 8 个沟通技巧

**预估时长：2.5 分钟**

8 个技巧，我快速过一下：

1. **先说背景，再提要求**——让 AI 先理解你的意图，正确率更高
2. **让 AI 先提问**——不要直接让它写，先让它问你问题
3. **先要方案，再要代码**——让 AI 提供几个方案，选定后再实现
4. **复杂需求拆小步**——一步一步来，满意就保存，逐步叠加
5. **初稿不求完美**——先生成再优化，迭代比完美更重要
6. **修改意见要具体**——说清楚哪里不对、想要什么效果
7. **规范提前约定**——技术栈、设计风格等，项目开始时就告诉 AI
8. **按任务分窗口**——不同功能开不同对话窗口，上下文更清晰
